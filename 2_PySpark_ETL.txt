# E-Commerce Transaction Analytics Pipeline
## PySpark ETL Solution for Cross-Border Sales Analysis

**Author:** Jarosław Błaziński  
**Project Type:** Portfolio Demonstration (Databricks-ready)  
**Tech Stack:** PySpark 3.x, Python 3.8+  
**Dataset Size:** ~500,000 transactions

---

## 📋 Business Problem

An e-commerce company operating across European markets (Poland, Germany, UK) faces challenges with their transaction data analysis:

### Key Issues:
1. **Data Quality Problems**
   - ~15% of transaction records contain NULL values in critical fields (transaction_id, amount)
   - Negative amounts (refunds/chargebacks) are mixed with regular sales, skewing revenue reports
   - Inconsistent data types preventing accurate calculations

2. **Manual VAT Calculations**
   - Finance team manually calculates VAT for different EU markets
   - Error-prone Excel-based processes
   - Difficult to audit and reconcile

3. **Lack of Performance Insights**
   - No automated way to identify top-performing categories per market
   - Management cannot quickly answer: "Which product categories drive revenue in each country?"
   - Delayed monthly reports (3-5 days manual processing)

4. **Scalability Issues**
   - Current Excel-based solution breaks with >100k rows
   - Growing to 500k+ transactions monthly
   - Need for big data processing capability

---

## 💡 Solution Overview

This project demonstrates a **production-ready PySpark ETL pipeline** that:

✅ **Cleanses** transaction data with automated quality checks  
✅ **Transforms** raw data into analytics-ready format with proper VAT calculations  
✅ **Enriches** transactions with country dimension data  
✅ **Analyzes** performance using advanced SQL operations (window functions, aggregations)  
✅ **Scales** to handle 500k+ records efficiently  

### Business Value:
- ⏱️ **Reduced processing time:** 3-5 days → **<15 minutes**
- 📊 **Automated insights:** Real-time ranking of top categories per country
- ✅ **Data quality:** Built-in validation and logging
- 🔍 **Audit trail:** Complete data lineage and quality metrics

---

## 🏗️ Technical Architecture

```
Raw Data (CSV)
     ↓
[Data Quality Assessment]
     ↓
[Data Cleaning Layer]
  - Remove NULL values
  - Filter invalid transactions
  - Type casting
     ↓
[Transformation Layer]
  - VAT calculation (23% Polish rate)
  - Dimension enrichment (country names)
  - Derived metrics
     ↓
[Analytics Layer]
  - Aggregations by country/category
  - Window functions (ranking)
  - Performance metrics
     ↓
[Output Layer]
  - Partitioned Parquet files
  - Quality reports
```

## 🛠️ How to Run

### Prerequisites:
- Python 3.8+
- PySpark 3.x
- Jupyter Notebook or Databricks workspace

### Option 1: Local Execution
```bash
# Install dependencies
pip install pyspark jupyter

# Launch notebook
jupyter notebook PySpark_ETL_Advanced.ipynb

# Run all cells (data is generated automatically)
```

### Option 2: Databricks
1. Import `PySpark_ETL_Advanced.ipynb` to Databricks workspace
2. Attach to cluster (DBR 11.3+ recommended)
3. Run all cells
4. Modify paths to use DBFS: `dbfs:/FileStore/...`

### Configuration:
Edit these parameters in the notebook:
```python
VAT_RATE = 0.23              # Polish VAT rate
USE_SAMPLE_DATA = True       # Set False to use external CSV
INPUT_PATH = "your/path"     # Optional: external data source
OUTPUT_PATH = "your/path"    # Output location
```
